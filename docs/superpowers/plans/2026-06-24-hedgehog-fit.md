# Hedgehog Fit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local-previewable TRMNL plugin that shows an 80s-fitness-video hedgehog with a cheeky German motivational saying on the full (800×480) e-ink view.

**Architecture:** A `static`-strategy TRMNL plugin developed with the `trmnlp` gem. Six hedgehog poses + their sayings + the form-of-address pools live as JSON in `static_data`. The six images are base64 data-URIs assigned in `shared.liquid` (generated by a script). `full.liquid` picks a pose and saying pseudo-randomly via a time-seeded index (TRMNL Liquid has no random filter), injects the configured form of address with the `replace` filter, and renders a Grid split layout (saying left, image right) plus a `title_bar`.

**Tech Stack:** Ruby 4.0.5 (pinned via `.ruby-version`), `trmnl_preview` gem (`trmnlp`), TRMNL Liquid + Design System framework classes, ImageMagick (`magick`) for image preprocessing.

## Global Constraints

- **Runtime:** Ruby **4.0.5**, pinned via `.ruby-version`. Do NOT use Docker. Do NOT use the broken rbenv 4.0.1.
- **Scope:** Only the **full** size (800×480). Other sizes get minimal stubs only.
- **No custom styles:** never use inline `style="..."` or `<style>` blocks — framework classes only (`layout`, `grid`, `col--span-*`, `title`, `label`, `image`, `title_bar`).
- **No emojis** anywhere in markup.
- **Images:** every content `<img>` carries `class="image image-dither"`. Source PNGs downscaled to ~400px wide grayscale, embedded as base64 data-URIs.
- **Do not wrap markup** in `<div class="view view--*">` — start directly with `layout`. URLs use `trmnl.com`.
- **Guard nil:** custom-field value defaults to `neutral`; guard arrays before indexing.
- **Language:** German only. Tone: cheeky 80s-aerobics trainer.
- Selection seed: `trmnl.system.timestamp_utc | date: "%s" | plus: 0`.
- Form-of-address pools — weiblich: `Süße, Liebes, Powerfrau`; männlich: `Großer, Champion, Kämpfer`. Sayings verbatim from the spec.

### Task 3: Generate base64 image array into shared.liquid

A script reads the six processed PNGs and writes `src/shared.liquid` with a single `hog_images` array of data-URIs, indexable by pose.

**Files:**
- Create: `bin/encode-images.sh`
- Modify (generate): `src/shared.liquid`

**Interfaces:**
- Consumes: `src/assets/igel-1.png`..`igel-6.png` from Task 2.
- Produces: in `shared.liquid`, a Liquid variable `hog_images` = array of 6 `data:image/png;base64,...` strings, where `hog_images[0]` is Planke … `hog_images[5]` is Bizeps. Consumed by `full.liquid` as `hog_images[pose_idx]`.

- [ ] **Step 1: Write the generator script**

`bin/encode-images.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
out="src/shared.liquid"
{
  echo '{%- comment -%} GENERATED by bin/encode-images.sh — do not edit by hand {%- endcomment -%}'
  printf '{%% assign hog_images = "'
  for i in 1 2 3 4 5 6; do
    [ "$i" -gt 1 ] && printf '|||'
    printf 'data:image/png;base64,'
    base64 -i "src/assets/igel-$i.png" | tr -d '\n'
  done
  printf '" | split: "|||" -%%}\n'
} > "$out"
echo "wrote $out ($(wc -c < "$out") bytes)"
```

- [ ] **Step 2: Make executable and run**

Run: `chmod +x bin/encode-images.sh && ./bin/encode-images.sh`
Expected: prints `wrote src/shared.liquid (... bytes)` — size in the low hundreds of KB.

- [ ] **Step 3: Verify the generated Liquid**

Run: `head -c 200 src/shared.liquid; echo; grep -c 'data:image/png;base64,' src/shared.liquid`
Expected: starts with the GENERATED comment + `{% assign hog_images = "data:image/png;base64,...`; the grep count is `1` (one line, six occurrences inside it). Confirm the line ends with `| split: "|||" -%}`.

### Task 5: Full view — selection logic, anrede injection, Grid split layout

The heart of the plugin. Replace `full.liquid` with the time-seeded selection, `{ANR}` replacement, and the Layout A Grid split.

**Files:**
- Modify: `src/full.liquid`

**Interfaces:**
- Consumes: `poses`, `anrede` (Task 4), `hog_images` (Task 3), `trmnl.plugin_settings.custom_fields_values.anrede`, `trmnl.system.timestamp_utc`.

- [ ] **Step 1: Write `src/full.liquid`**

```liquid
{%- assign seed = trmnl.system.timestamp_utc | date: "%s" | plus: 0 -%}
{%- assign pose_count = poses | size -%}
{%- assign pose_idx = seed | modulo: pose_count -%}
{%- assign pose = poses[pose_idx] -%}
{%- assign saying_count = pose.sayings | size -%}
{%- assign say_idx = seed | divided_by: 7 | modulo: saying_count -%}
{%- assign saying = pose.sayings[say_idx] -%}

{%- assign anrede_value = trmnl.plugin_settings.custom_fields_values.anrede | default: "neutral" -%}
{%- assign anr = "" -%}
{%- unless anrede_value == "neutral" -%}
  {%- assign pool = anrede[anrede_value] -%}
  {%- if pool and pool.size > 0 -%}
    {%- assign anr_idx = seed | divided_by: 13 | modulo: pool.size -%}
    {%- assign anr = ", " | append: pool[anr_idx] -%}
  {%- endif -%}
{%- endunless -%}
{%- assign text = saying | replace: "{ANR}", anr -%}

<div class="layout">
  <div class="grid">
    <div class="col--span-7">
      <div class="layout layout--col layout--center-y gap">
        <span class="title title--large" data-fit-value="true">{{ text }}</span>
        <span class="label label--small label--gray">{{ pose.label }}</span>
      </div>
    </div>
    <div class="col--span-5">
      <div class="layout layout--center">
        <img class="image image-dither" src="{{ hog_images[pose_idx] }}">
      </div>
    </div>
  </div>
</div>

<div class="title_bar">
  <img class="image" src="https://trmnl.com/images/plugins/trmnl--render.svg">
  <span class="title">{{ trmnl.plugin_settings.instance_name | default: "Hedgehog Fit" }}</span>
  <span class="instance">Fitness-Igel</span>
</div>
```
